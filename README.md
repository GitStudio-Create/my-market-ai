# My Market AI

株価・指数・ETF・ニュースと自分のメモをまとめ、考えた理由や確認事項を時系列で振り返るための投資情報整理ノートです。HTML/CSS/JavaScriptだけで動作します。

## 目的

My Market AIは売買アプリではありません。市場情報を見て、そのとき気になった理由や確認したい事実を記録し、後から振り返るための個人用ノートです。

## 主な機能

- 任意の銘柄コード・表示名・種類を登録できるウォッチリスト
- 個別株・指数・ETFに対応した日本／米国の入力候補
- API接続可能な現在価格、前日比、直近30秒のCanvas簡易チャート
- 配信元・時刻・別タブリンク・データ種別付きの関連ニュース
- ニュース見出しの共通テーマと重複を整理する事実ベースのAI要約表示
- 銘柄ごとのウォッチ理由、自由タグ、お気に入り
- 日付と分類を付けて何件でも保存できるメモ
- 「なぜ気になったか」「確認したこと」「次に確認したいこと」「決算」「配当」「ニュース」「その他」の分類
- 件数・最終メモ日・分類フィルター付きのメモ履歴
- JSONバックアップ・復元、UTF-8 BOM付きCSV出力
- 免責事項・プライバシーポリシー・利用規約の案内ページ
- 補助的な価格・変動率アラート
- localStorageによる登録対象とメモ履歴の保存
- ライト／ダークモード、PC 2列／スマートフォン1列表示

## 既存データの移行

旧Stock Alert Memoで保存した登録対象はそのまま読み込まれます。旧形式の単一メモは、初回読み込み時に日付付きの「その他」メモとして履歴へ移行されます。既存の旧分類名が付いた履歴は書き換えず、分類フィルターの候補へ追加して表示します。

## 使い方

`index.html` をブラウザで開くだけで動作します。ビルド処理と外部ライブラリは不要です。中継APIを設定しない場合は、価格・チャート・ニュース・AI要約がサンプル用データで動作します。

## 現在のデータモード

現在は `script.js` 冒頭の `DATA_MODE = 'api'` とCloudflare Workerの `API_PROXY_BASE_URL` を設定し、価格とチャートを実データで取得します。ニュースはStage12時点では安全なサンプルJSONです。

中継APIが未設定または取得に失敗した場合は、対象データだけサンプル表示へ安全にフォールバックします。

## Stage11：実データAPI接続準備

GitHub Pagesから外部サービスへ直接APIキーを送らず、サーバーレス関数などの中継APIを経由する構造へ変更しました。価格・チャートはTwelve Data、ニュースはFinnhub、Alpha Vantage、NewsAPI、Twelve Data Newsなどを中継側で選択できます。

1. サーバーレス関数側の環境変数へ、利用するデータ提供元のAPIキーを登録します。
2. 中継APIに `market`、`chart`、`news` の各エンドポイントを実装します。
3. `script.js` 冒頭の `API_PROXY_BASE_URL` に中継APIの公開URLだけを設定します。
4. `DATA_MODE = 'demo'` を `DATA_MODE = 'api'` に変更します。

フロントエンドは次のリクエストを送ります。APIキーをクエリ、ヘッダー、レスポンスへ含めないでください。

- `GET {API_PROXY_BASE_URL}/market?symbol=AAPL`
- `GET {API_PROXY_BASE_URL}/chart?symbol=AAPL&range=1m&interval=1day&outputsize=30`
- `GET {API_PROXY_BASE_URL}/news?symbol=AAPL`

応答は `{ "data": ... }` で包むか、データ本体を直接返せます。価格は `price`、チャートは `points` または `values`、ニュースは配列または `items` / `articles` / `news` 配列を利用します。提供元固有の応答は、できるだけ中継API側でこの共通形式へ変換してください。

API取得失敗時は対象データだけサンプルへフォールバックし、価格、チャート、ニュース、メモのほかの機能は動作を続けます。`example.com` のニュースとフォールバック記事には「サンプル」、実ニュースURLには「実ニュース」と表示し、どちらも安全な新しいタブで開きます。

> **重要:** GitHub Pagesは静的サイトのため、ブラウザへ配信した秘密情報を保護できません。APIキーは公開リポジトリ、`script.js`、フロントエンドの環境変数へ記載せず、サーバーレス関数側の暗号化された環境変数で管理してください。

## Stage12：Cloudflare Workers 中継API準備

リポジトリ直下の `worker.js` はCloudflare WorkersのModule Workerとして動作します。Twelve DataのAPIキーはコードへ含めず、Worker Secret `TWELVE_DATA_API_KEY` から読み取ります。

### Cloudflare Dashboardでの設定

1. Cloudflare Dashboardの「Workers & Pages」からWorkerを作成します。
2. Workerの編集画面へ `worker.js` の内容を貼り付けてデプロイします。
3. Workerの「Settings」→「Variables and Secrets」で `TWELVE_DATA_API_KEY` を追加し、種類を **Secret** にしてTwelve DataのAPIキーを保存します。
4. 必要に応じて `ALLOWED_ORIGINS` を通常の環境変数として追加します。複数指定はカンマ区切りです。
5. 発行された `https://<worker-name>.<account>.workers.dev` を `script.js` の `API_PROXY_BASE_URL` に設定します。
6. `DATA_MODE` を `'api'` に変更し、GitHub Pagesへ反映します。

`ALLOWED_ORIGINS` を省略した場合は、このGitHub Pages公開元とローカル確認用Originだけを許可します。独自ドメインを使用する場合は、例のように明示してください。

```text
ALLOWED_ORIGINS=https://example.com,https://www.example.com
```

CORSはブラウザからの利用元を制限する仕組みであり、APIそのものの認証ではありません。公開後はCloudflareの利用状況とTwelve Dataのレート制限を確認し、必要に応じてCloudflareのRate Limitingも追加してください。`ALLOWED_ORIGINS=*` は通常の公開運用では推奨しません。

Wranglerを使用する場合も、APIキーは設定ファイルの `vars` へ書かず、Secretとして登録してください。

```shell
npx wrangler secret put TWELVE_DATA_API_KEY
npx wrangler deploy worker.js --name my-market-ai-api
```

ローカル開発で `.dev.vars` または `.env` を使用する場合、これらは `.gitignore` の対象です。実キーをGitへ追加しないでください。

### Workerのエンドポイント

- `/market?symbol=AAPL`：Twelve Data `/quote` から現在価格、前日比率、通貨、取得時刻を返す
- `/chart?symbol=AAPL&range=1d`：5分足78件
- `/chart?symbol=AAPL&range=1w`：1時間足35件
- `/chart?symbol=AAPL&range=1m`：日足30件
- `/news?symbol=AAPL`：実ニュースAPI接続前の安全なサンプルJSONを返す
- `/health`：Workerの稼働確認用

すべての応答はJSONです。入力値、許可Origin、HTTPメソッドを検証し、エラー時も `{ "error": { "code": "...", "message": "..." } }` 形式で返します。APIキーやTwelve DataへのリクエストURLは応答へ含めません。価格・チャート・ニュースは情報整理用であり、投資助言、売買推奨、将来予測、株価予想には使用しません。

## Stage7：チャート本物化

Stage7ではTwelve Dataの時系列を利用できる構造を追加しました。Stage11以降の公開版では、同じ取得条件を中継APIへ渡し、APIキーをブラウザへ公開しない構成です。

- 1日：5分足、78件
- 1週間：1時間足、35件
- 1か月：日足、30件

同じ銘柄・期間のAPI結果は5分間キャッシュし、期間ボタンの連打による不要なAPI消費を抑えます。中継API未設定、通信失敗、APIエラー、終値データが2点未満の場合は、同じ銘柄・期間のサンプルチャートへ自動的に戻ります。フォールバック時はチャートに `SAMPLE FALLBACK` と表示します。

APIから返された系列とデモ系列は、どちらも `{ symbol, range, source, points }` の共通形式で `renderChart()` に渡されるため、既存の線・価格ラベル・余白デザインを維持します。

## Stage8：AI要約強化

固定文だったAI要約を、取得済みニュースのタイトルと種類ラベルから自動生成するルールベース要約へ変更しました。`generateAISummary(newsItems)` が次の順で3〜5行程度に整理します。

- URLまたは正規化したタイトルが同じ重複記事をまとめる
- 決算・業績、製品・サービス、経済指標、ETF、市場全体・指数、企業発表・事業動向の共通テーマを抽出する
- 記事数が多いテーマと、そのほかに含まれるテーマを事実ベースで記述する
- ニュースが2件未満の場合は「十分なニュースがありません」と表示する
- 投資判断や将来予測を目的としないことを要約内に明記する

要約生成はニュース一覧の描画後に独立した例外処理で実行します。要約処理でエラーが起きても、ニュースのリンク・配信元・時刻・種類表示には影響せず、要約欄だけが専用のエラーメッセージへ切り替わります。

将来AI APIへ接続する場合は、ニュース取得や画面描画を変更せず、`generateAISummary(newsItems)` の内部を安全な中継API呼び出しへ差し替える構成です。

## Stage8.5：ウォッチリスト自由追加

既存の登録フォームを拡張し、固定候補にない銘柄コードもウォッチリストへ追加できるようにしました。

- 銘柄コード、任意の表示名、種類、最初のメモを入力可能
- 種類は「個別株」「ETF」「指数」「その他」から選択
- 表示名が空欄の場合は候補データの名前、候補もない場合は銘柄コードを使用
- 銘柄コードは入力時と保存時に大文字へ統一
- 空欄・使用できない文字・大文字小文字を問わない重複登録を防止
- 追加直後に既存カードと同じUIへ反映し、`localStorage`へ保存
- 再読み込み後も登録対象、メモ履歴、種類などを維持
- 既存の削除ボタンで、追加した対象とそのメモ履歴を削除可能

候補外の銘柄も、サンプル価格・サンプルチャート・種類と地域に応じたサンプルニュース・AI要約・メモを表示します。APIモードでは中継APIへ入力されたシンボルを渡し、中継API未設定や取得失敗時はサンプルデータへ戻ります。ひとつのデータ取得に失敗しても、ニュースやメモなどほかの機能は動作を続けます。

保存データには今後の拡張用として、従来の `symbol`、`name`、`type`、`notes` に加え、`assetClass`、`memo`、`favorite`、`tags` を保持します。以前の保存データは読み込み時に不足項目を補うため、保存キーを変更せずに引き続き利用できます。

> **公開時の注意:** APIキーはフロントエンドのJavaScriptへ直書きせず、サーバーレス関数などの中継APIで環境変数として管理してください。

## Stage9：投資ノート強化

各カードへ、あとから振り返るための整理機能を追加しました。

- ウォッチ理由：自由入力した理由を銘柄ごとに保存
- タグ：自由入力のタグを最大12件まで追加し、カード上部にも表示
- お気に入り：カード右上の `☆ / ★` で切り替え、カード枠でも状態を識別
- メモ分類：「なぜ気になったか」「確認したこと」「次に確認したいこと」「決算」「配当」「ニュース」「その他」
- 履歴フィルター：分類ごとに絞り込み、該当件数と全件数を表示
- メモ統計：メモ総数と最終メモ日をカード内に表示
- 履歴表示：新しい順を維持し、履歴部分をスクロール可能な高さに制限

ウォッチ理由とタグ編集欄は折りたたみ式です。お気に入りと最大3件のタグはカード上部へ常時表示し、情報を増やしてもカードが縦に長くなりすぎない構成にしています。

保存データには `watchReason: ""`、`tags: []`、`favorite: false`、`notes: []` を保持します。古い保存データに項目がない場合は `normalizeStoredStock()` が初期値を補完するため、従来の登録対象・メモ・価格表示を維持したまま移行できます。フィルターは表示状態のみで、メモ本文や分類を書き換えません。

## Stage9.5：データ出力・バックアップ機能

登録フォームの上に折りたたみ式の「データ管理」を追加しました。バックアップファイルには個人的なメモが含まれるため、安全な場所へ保管してください。

### JSONバックアップ

`my-market-ai-backup-YYYYMMDD.json` として、次の内容を保存します。

- バックアップ形式のバージョンと出力日時
- ライト／ダークのテーマ設定
- ウォッチリスト、表示名、種類、ウォッチ理由、タグ、お気に入り
- 各銘柄のメモ履歴と既存表示に必要なデータ

### JSONから復元

新しいバックアップ形式に加え、配列だけの旧localStorage形式、`stocks` または `data` 配列を持つ旧形式も読み込めます。復元前に次の検証を行います。

- JSONとして解析できること
- ウォッチリスト配列と有効な銘柄コードがあること
- 同じ銘柄コードが重複していないこと
- ファイルサイズが5MB以下、登録対象が500件以下であること

検証に成功した場合だけ「現在の保存データを上書きします」という確認ダイアログを表示します。キャンセルまたは検証失敗時は既存データを変更しません。復元時は `normalizeStoredStock()` と `normalizeStoredNote()` が不足項目や古いメモ形式を補完し、保存後に価格・チャート・ニュース・AI要約を再描画します。

### CSV出力

メモ履歴を `my-market-ai-notes-YYYYMMDD.csv` として出力します。列は `symbol`、`name`、`type`、`favorite`、`tags`、`watchReason`、`noteDate`、`noteCategory`、`noteText` です。Excelで開きやすいようUTF-8 BOMとCRLF改行を使用し、タグは ` | ` 区切りで格納します。メモがない登録対象も、メモ列が空の1行として出力します。

## データ取得層

画面描画とデータ取得を分離するため、次の共通関数を設けています。

- `fetchMarketData(symbol)`：現在価格、前日比、通貨、取得時刻、データ提供元
- `generateDemoChartData(symbol, range)`：銘柄・期間別のデモ価格配列を生成
- `fetchRealMarketData(symbol)`：中継APIから現在価格の共通形式を取得
- `fetchRealChartData(symbol, range)`：中継APIから価格系列の共通形式を取得
- `fetchRealNewsData(symbol)`：中継APIから実ニュースを取得して共通形式へ変換
- `fetchChartData(symbol, range)`：API取得とデモフォールバックを統括
- `renderChart(symbol, chartData, card)`：取得済みデータをCanvasへ描画
- `generateDemoNewsData(symbol)`：対象の種類・地域に合わせたサンプルニュースを生成
- `fetchNewsData(symbol)`：見出し、配信元、時刻、種類ラベル、URLを取得
- `renderNews(symbol, newsData, card)`：取得済みニュースをリンク付きで描画
- `generateAISummary(newsItems)`：ニュース見出しの重複と共通テーマを整理した事実ベース要約

現在価格、チャート、ニュースは `DATA_MODE` に対応するデータ提供元を呼び出します。APIモードでは `dataProviders.api` が3つの実データ関数へ処理を委譲し、デモモードではサンプル提供元を利用します。画面側は同じ形式の戻り値を利用するため、API提供元を変更しても表示処理を変更する必要がありません。

## 将来のAPI連携

`dataProviders.api.fetchMarketData`、`fetchChartData`、`fetchNewsData` は、それぞれ `fetchRealMarketData`、`fetchRealChartData`、`fetchRealNewsData` へ接続しています。これらは中継APIを通じて取得し、画面が利用する共通形式へ変換します。ニュースのAI要約は、取得元にかかわらずブラウザ内で見出しを事実ベースに整理します。

ニュース提供元は中継API側で差し替えます。Finnhub、Alpha Vantage、NewsAPI、Twelve Data Newsなどの応答を共通ニュース形式へ変換し、記事URLは必ずAPI提供元が返した正規のURLを使用してください。実ニュースは「実ニュース」、`example.com` またはフォールバック記事は「サンプル」と表示されます。

検索候補は `stockDataProvider` の `search()` と `getBySymbol()` を検索APIへ置き換えられます。各候補は種類、地域、市場、更新方式を持ちます。投資信託向けには `assetTypeSettings` に日次更新方式を定義していますが、現時点では候補データに含めていません。

APIキーをブラウザ側へ直接埋め込まず、安全な中継処理を利用してください。

## GitHub Pages

`index.html`、`disclaimer.html`、`privacy.html`、`terms.html`、`style.css`、`script.js`、`legal.js` を同じ公開ディレクトリへ置き、GitHub Pagesの公開元を対象ブランチのルートへ設定してください。すべて相対パスのため、サブパスで公開されるプロジェクトページでもページ間を移動できます。

公開前に次を確認してください。

- 秘密のAPIキーをフロントエンド、公開リポジトリ、Git履歴へ含めない
- `API_PROXY_BASE_URL` には公開可能な中継API URLだけを設定する
- 実APIを利用する場合は、環境変数でキーを管理できるサーバーレス関数等の中継APIを使用する
- localStorageのデータは利用者のブラウザ内に保存され、キャッシュ削除や端末変更で消える可能性があることを案内する
- 価格、チャート、ニュース、AI要約が情報整理用であり、投資助言・売買推奨・将来予測ではないことを維持する
- 公開時の実際の仕様に合わせて、3つの案内ページを必要に応じて見直す
